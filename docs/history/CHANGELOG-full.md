## How to add an entry (read before you log)
Add the new entry at the **TOP** of the list below (newest first). One bullet per build/change:

```
- **v0.105 · build cut — packages this session's work; tools renamed v0.104→v0.105** (`DATA.version` stays **v0.322** — no price/ladder change this session; **46 gates green**, build-check **G1** + version-consistency **G3** OK). Packages: the **unified DM console** (CardGrid+DataTable merged → `Unified-v0.015`, originals retired), engine deltas (compute **Ki-unused advisory**; new **`DATA.masteryFx`** with 8 weapon-mastery descriptions), the **F3** swap rule (free cross-level swap = difference, paid = full new cost), the LiveSheet **centre-panel** additions (ability/boon tooltips; Arts/Maneuvers/Metamagic/Weapon-Mastery segments; Vigor/Grit/Attunement tiles — all page-ready), and the CharGen/LiveSheet **dark-mode + label** fixes. Open: PHB page numbers + 69 drawback descriptions (task #13, awaiting source).
- **vX.YYY · <type> — <one-line headline>** (<proof: N gates green, build-check OK, guide-reconcile
  drift, files touched>). <What changed and why, in condensed prose.>
```

- **`<type>`** is one of: `build cut` · `feature` · `rule` · `fix` · `UI` · `polish` · `tooling`.
- Note **`DATA.version`** only when it changed (`→ vX.YYY`); for build/tooling-only changes say `stays vX.YYY`.
- State the **proof**: gate count from `node tests/audit-all.cjs`, **build-check (G1)** status (tools = `src/engine/`),
  and guide-reconcile drift if the guide was touched.
- **The G6 gate fails** if you bump `DATA.version` without an entry here that mentions that version — so log first.
- Keep it to the condensed one-paragraph style; verbose reasoning / dead-ends go in `archive/`.
- After logging, also refresh `RESTART-STATUS-v0.322.md` and the `PACT-CURRENT` markers (INDEX + CONTEXT).
